## 短版 (300 字内 · 项目卡片摘要)

**VBTI(Voice-BS Type Indicator,声音照妖镜)** 是一款移动端 Web 声音测试类应用. 60 秒 · 5 个案发现场 · 1 张可分享的判决书.

不同于 MBTI/SBTI 靠自选答案的心理测验,VBTI **只听声音**: ASR 转文本后,LLM 判"内容有多激动"(语义 arousal), DSP 从音频抽 F0 起伏 / 音量动态 / 峰值密度(声学 arousal), 两者之差就是 **反差率** —— 你嘴上稳不稳、声音在不在报警. 再叠加抓马浓度 + 语速稳定性等指标, 匹配到 **5 语系 × 7 人格 = 35 张开放卡池**(影视组/综艺组/舞台组/机器人组/街头组).

结果页做成 **档案室判决书**: 5 项声音暴露指数 · 3 条罪证证据 · "已实锤"红印章 · 分享给同担, 支持一键"公开处刑". 视觉是 dark detective / 铁锈红 + 铜金 / 做旧纸卡, 反默认 AI 紫色美学.

**技术**: Next.js 14 · TypeScript · faster-whisper CPU 推理 · GPT-5 情绪抽取 · YIN F0 + Meyda RMS + 能量 VAD + 3 新特征 · 数据驱动开放卡池 · Prisma SQLite. 完全开源, Docker 一键部署.

## 长版 (800-1000 字 · 完整介绍)

### 项目名
**VBTI · 声音照妖镜** (Voice-BS Type Indicator)

### 一句话定位
说 60 秒话, VBTI 用**反差率**和**抓马浓度**两个可测声学指标, 把你钉在一张演艺人格卡上——你不是没在演, 只是没意识到自己演得像谁.

### 想解决什么

MBTI / SBTI 靠自选答案, 用户能主动包装人设. 但**声音在高压/破防/入戏的时候会泄露**更多真实状态: 语速抖不抖、停顿多不多、音高有没有飙、"绝了/笑死"这类词往外冒的频率——这些用户自己听不见, 但系统一测就知道.

VBTI 抓的正是这个"演的痕迹". 它不给你贴 MBTI/性格标签, 只用**可测的声学证据**告诉你: 你在扮什么角色, 演得像谁, 反差有多大.

### 核心玩法

**60 秒 · 5 个案发现场**:
- 案发现场 01 · 机会争夺战 · "为什么我们要选你?"
- 案发现场 02 · DDL 救火 · "你先帮我顶一下吧"
- 案发现场 03 · 公开甩锅 · "这块是 ta 负责的"
- 案发现场 04 · 本命塌房 · 给同担吐槽官方把角色写崩
- 案发现场 05 · 三秒入戏 · 连演反派 / 治愈系 / 中二三个角色

前 3 题压反差(嘴上稳、声音破防), 第 4 题拉抓马浓度(唯一激活点), 第 5 题测入戏速度和变声跨度.

### 技术核心

**两条可测信号的差, 不是玄学**:

```
反差率 = | 语义激动度 − 声学激动度 | × 100
       ∈ [0, 100]

  语义激动度 · LLM 从 ASR 文本判"内容有多激动"     ∈ [0,1]
  声学激动度 · normalize(F0_std + RMS_dr + 语速 +
              峰值密度) 权重求和                    ∈ [0,1]

抓马浓度 = normalize(F0_std·0.35 + RMS_dr·0.35 +
                    peak_density·0.30) × 100
```

**为什么单轴 arousal 而不是二维 valence+arousal**:
声学信号能可靠反映激动度, 但**测不出正负情绪** (同样的高音量, 可能狂喜也可能暴怒). 拿不可靠的声学 valence 去比语义 valence, 得到的反差率大部分是噪声. VBTI 只做 arousal 反差, 保证每一个数字都有硬证据.

### 匹配算法

5 维曼哈顿距离 + 5 语系印记扳机:

```
5 语系 中心坐标 · 影视/综艺/舞台/机器人/街头
5 特征触发条件(如"高抓马外放"→街头组)
用户向量 = (反差率, 抓马浓度, 语速稳定, 音量强度, 独白倾向)
argmax(1 - 距离 + 触发加分)
```

**开放卡池设计**: 35 张起步(5 语系 × 7 人格), 加一张新人格是纯数据操作——改 `personas.ts` 加一条 config + 补一张 PNG, 匹配算法不变、数据结构不变、UI 不变.

### 差异化亮点

1. **声学证据可展开**: 每一项分数都能点开看真实特征值 ("语速 4.2 字/秒", "抓马 82%"), 反玄学.
2. **反差率专利式设计**: LLM 的语义打分 + DSP 的声学打分同时可靠,它们的差才是真信号.
3. **档案室审判美学**: 反 AI 默认紫色, 走 dark detective —— 铁锈红 + 铜金 + 做旧纸卡 + 台灯聚光 + "已实锤"红印章.
4. **35 张开放卡池**: 覆盖 5 语系 × 7 人格(方法派、综艺废话、导航播报员、麦霸等), 每张卡带手写文案 + gpt-image-2 立绘 + 声学中心坐标.
5. **社交传播闭环**: 揭晓页只给语系吊胃口 → 分享解锁完整判决书 → 卡片带二维码 → 扫码回流.

### 技术栈

| 层 | 选择 |
|---|---|
| App | Next.js 14 (App Router) + TypeScript + Tailwind CSS |
| DB | Prisma + SQLite |
| ASR | faster-whisper `small` int8, FastAPI sidecar, CPU 推理 ~2s/段 |
| LLM | OpenAI chat-completions 兼容 (gpt-5-chat-latest), 自动重试 + 关键词兜底 |
| DSP | 纯 Node: ffmpeg 解码 + YIN F0 + Meyda RMS + 能量 VAD + peak_density + pause_regularity + burst_stops = 17 维 |
| 匹配 | 5-D Manhattan + 5 印记扳机 + 稀缺/兜底/常规最近邻 |
| 立绘 | 35 张离线生成 (gpt-image-2), 请求路径零 API 调用 |
| 分享 | SVG 判决书 + QR 码 |
| 部署 | Docker Compose, 2 vCPU / 4GB RAM 即可 |

### 数据说明

**冷启动测试** (Phase 0 反差率可行性 spike): 8 段真实中文音频, 3 次重跑, 反差率均值 spread 21-50, 稳定性 std < 4 —— 声学信号在同一录音上具备可复现性, 不同录音间有明显区分度, 支持匹配到不同语系.
